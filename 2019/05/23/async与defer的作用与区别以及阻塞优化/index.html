<strong>css阻塞有什么解决方法？</strong></h6><p>css的加载并不会导致html解析和渲染的停止，但是会影响到js脚本的执行。因为js脚本不仅可以读取修改到dom，也可以读取修改到cssom。故在js脚本执前，browser必须保证到css文件完全加载并解析完成，即cssom树完全构建好。这就导致了js执行的延迟，也因此导致html解析和渲染延迟。（这就是css阻塞js执行，阻塞渲染的根本原因）<br>因此一些解决方法：</p> <ul> <li>1、在引入顺序上，css资源的引入要优于js脚本的引入</li> <li>2、对css进行精简并尽快提供</li> <li>3、可以用媒体类型（会加载不会阻塞）</li> <li>4、用媒体查询（会记载，只有在符合的设备上才会进行阻塞）</li> </ul> <p>参考文章：</p> <ul> <li><a
